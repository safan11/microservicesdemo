
API GATEWAY NOTES

1. What is API Gateway?
- API Gateway is a single entry point for all client requests in a microservices architecture.
- It routes requests to the appropriate microservice, applies filters, handles authentication, logging, and monitoring.
- It simplifies client interaction by hiding the complexity of multiple microservices.

2. Why We Need API Gateway
- Centralized routing: Clients don’t need to know addresses of individual services.
- Load balancing: Distributes requests across service instances.
- Security: Can enforce authentication/authorization at the gateway level.
- Monitoring & Logging: Provides a central place to log and monitor traffic.
- Protocol translation: Converts client requests (HTTP, WebSocket) into formats microservices understand.
- Simplifies client: Instead of calling multiple services directly, client calls only the gateway.

3. Your Gateway Configuration
- server.port: 8060 → Gateway runs on port 8060.
- eureka.client.serviceUrl.defaultZone: http://localhost:8061/eureka/ → Gateway registers with Eureka Service Registry.
- spring.cloud.gateway.discovery.locator.enabled: true → Enables automatic route discovery from Eureka.
- Routes defined manually:
  - id: helloworld-service
    uri: lb://helloworld-service → Load-balanced URI, resolved via Eureka.
    predicates: Path=/helloworld/** → Matches incoming requests with path /helloworld.
    filters: RewritePath=/helloworld/(?<path>.*), /${path} → Removes prefix before forwarding.
  - Similar routes defined for helloworld2-service and FSD.

4. Logging
- logging.pattern.console: Custom format for console logs with timestamp, log level, and message.

5. Swagger Integration (springdoc)
- springdoc.swagger-ui.urls: Defines multiple Swagger UI endpoints for different services.
- Example: 
  - name: helloworld
  - url: /v3/api-docs/helloworld
- This allows API documentation for each microservice to be accessible via the gateway.

6. Code Explanation (GroupedOpenApi Bean)
```
@Autowired
RouteDefinitionLocator locator;

@Bean
public List<GroupedOpenApi> apis() {
    List<GroupedOpenApi> groups = new ArrayList<>();
    List<RouteDefinition> definitions = locator.getRouteDefinitions().collectList().block();
    assert definitions != null;
    definitions.stream()
        .filter(routeDefinition -> routeDefinition.getId().matches(".*-service"))
        .forEach(routeDefinition -> {
            String name = routeDefinition.getId().replaceAll("-service", "");
            groups.add(GroupedOpenApi.builder()
                .pathsToMatch("/" + name + "/**")
                .group(name)
                .build());
        });
    return groups;
}
```

- RouteDefinitionLocator: Fetches all route definitions from the gateway.
- The code loops through all routes whose IDs end with "-service".
- For each route:
  - It extracts the service name (removes "-service").
  - Creates a GroupedOpenApi object for Swagger documentation.
  - Example: For "helloworld-service", it creates a Swagger group "helloworld" with paths "/helloworld/**".
- Purpose: Automatically generate Swagger groups for each microservice route, so documentation is organized per service.

7. Dependencies Required
- spring-cloud-starter-gateway → For API Gateway functionality.
- spring-cloud-starter-netflix-eureka-client → For service discovery with Eureka.
- springdoc-openapi-ui → For Swagger/OpenAPI integration.
- spring-boot-starter-actuator → For monitoring endpoints.

8. Flow
- Client sends request → Gateway receives it on port 8060.
- Gateway checks route definitions (manual + Eureka discovery).
- Applies filters (e.g., RewritePath).
- Forwards request to appropriate microservice via load-balanced URI.
- Swagger groups are auto-generated for each service route, making API docs accessible.

Summary:
API Gateway centralizes routing, security, and monitoring in microservices. Your configuration registers with Eureka, defines routes, rewrites paths, and integrates Swagger. The GroupedOpenApi bean dynamically creates documentation groups for each service route, simplifying API exploration.
